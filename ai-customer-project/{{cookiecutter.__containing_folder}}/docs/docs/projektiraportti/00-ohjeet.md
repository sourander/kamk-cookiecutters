# OHJEET PROJEKTIRAPORTTIIN

- Projektiraportin rakenne on viitteellinen: muokkaa raportin otsikoita ja sisältöä projektin tehtävänantoa dokumentointia palvelevaksi.
- Mikäli projektiraportissa on käytetty ulkopuolisia lähteitä, lähdeviitteisiin ja lähdeluetteloon käytetään OAT-specia (_**kts. [ohjeet](https://sourander.github.io/oat/weeks/05_lahdeviittaukset/)**_)
- Mikäli haluat tarkistaa lokaalisti, miltä projektiraporttisi näyttää, voit tehdä sen suorittamalla komennot:
    ```bash
    cd docs/
    uvx --with mkdocs-material --with mkdocs-awesome-nav mkdocs serve --livereload --open --strict
    ```
